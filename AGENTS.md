# AGENTS.md

## Project Overview

**Type**: sveltekit + SvelteKit 2 + Svelte 5 + Tailwind CSS v4
**Runtime**: Cloudflare Workers
**Package Manager**: pnpm 11.2.2

## Development Environment

A Nix flake is present for local dev. If using direnv, it loads automatically:

```bash
direnv allow
```

Otherwise:

```bash
nix develop
```

## Commands

- `pnpm dev`: Start Vite dev server
- `pnpm build`: Production build
- `pnpm preview`: Build + `wrangler dev` (test Worker locally)
- `pnpm check`: Typecheck (`svelte-kit sync` + `svelte-check`)
- `pnpm check:watch`: Typecheck in watch mode
- `pnpm lint`: ESLint
- `pnpm lint:fix`: ESLint with auto-fix
- `pnpm format`: Prettier check
- `pnpm format:fix`: Prettier write
- `pnpm gen:cf-types`: Regenerate worker types from `wrangler.jsonc`

## CI Pipeline

GitHub Actions runs independent jobs on PRs/pushes to `main`/`develop`:

1. `format` — Prettier formatting
2. `lint` — ESLint with TypeScript, Svelte and Perfectionist
3. `typecheck` — `svelte-check` (requires `gen:cf-types` first)
4. `build` — Production build

All jobs use `pnpm install --frozen-lockfile`.

## Architecture Notes

- **Adapter**: `@sveltejs/adapter-cloudflare`
- **Entry**: `wrangler.jsonc` configures the Worker
- **Generated Types**: `src/worker-configuration.d.ts` is auto-generated by `wrangler types`. Rerun `pnpm gen:cf-types` when changing `wrangler.jsonc`.

## Code Style

**Prettier**: 4-space indentation, double quotes, trailing commas. YAML uses 2-space tabs.

**ESLint**: TypeScript strict mode, Svelte all rules enabled, plus Perfectionist for alphabetical sorting of imports/exports/objects.

**Svelte**: All `.svelte` files must use `<script lang="ts">` (enforced by `svelte/block-lang`).

**Imports**: Alphabetical order within groups. Use `// @sort` comments to partition sorting.

## Important Files

- `wrangler.jsonc`: Worker config: bindings, compatibility flags, routes
- `svelte.config.js`: SvelteKit configuration
- `vite.config.ts`: Vite plugins and build config
- `tsconfig.json`: TypeScript compiler options
- `eslint.config.js`: ESLint rules and plugins
- `.prettierrc`: Prettier formatting config
- `flake.nix`: Nix development shell

## Deployment

Cloudflare Workers via Wrangler. The demo/site is deployed through the Cloudflare dashboard, not via CI.
